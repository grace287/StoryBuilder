# 스크리브너(Scrivener) 벤치마킹

> StoryBuilder 기획·기능 설계 참조 문서. 스크리브너는 '끝판왕' 작가 툴로 불리는 **집필 관리 시스템**이다.

---

## 1. 스크리브너 핵심 기능 및 사용법

### ① 바인더 (Binder) — 파일 관리

- 화면 왼쪽 **바인더**: 한 권의 책을 챕터·장면(Scene) 단위로 잘게 쪼개어 관리.
- **사용법**: 긴 글을 한 파일에 쓰지 않고, 장면마다 파일을 만든다. 순서 변경은 바인더에서 **드래그 앤 드롭**으로 처리.

**StoryBuilder 반영**: 계층형 바인더 (Project → Chapter/회 → Scene), `order_index` 기반 순서 변경, 드래그 앤 드롭 API (`PATCH /chapters/reorder`, `PATCH /scenes/reorder`).

---

### ② 코르크 보드 (Corkboard) — 시각적 기획

- 각 파일(장면)을 **인덱스 카드**로 표시.
- **사용법**: 카드에 요약을 적어 두고 전체 흐름을 한눈에 파악하며 플롯 구성. 카드 이동 시 원고 순서도 연동.

**StoryBuilder 반영**: 씬/챕터에 `title`, `summary` 필드. 카드 뷰는 프론트에서 동일 데이터로 구현 가능.

---

### ③ 스플릿 뷰 (Split View) — 참조와 집필 동시

- 화면을 가로/세로 분할.
- **사용법**: 한쪽에 참조 자료(PDF, 이미지, 웹), 다른 쪽에 원고. 창 전환 없이 참조하며 집필.

**StoryBuilder 반영**: MVP 로드맵 Phase 1.2 스플릿 뷰 (참조 문서 + 현재 씬 동시 표시).

---

### ④ 컴파일 (Compile) — 원클릭 출판

- 수많은 조각 글을 하나로 합쳐 **ePub, PDF, Word** 등으로 내보내기.
- **사용법**: 서식 설정 후 출판 규격 원고 자동 생성.

**StoryBuilder 반영**: 로드맵 Phase 2/3 — 출판 포맷 변환, 내보내기 API.

---

### ⑤ 스냅샷 (Snapshot) — 버전 보관

- 퇴고 전 버전을 저장. 수정 후 비교·복구 가능. (개발자의 **Git**과 유사.)

**StoryBuilder 반영**: `Manuscript` 테이블에 버전 관리. Redis/DB 스냅샷 정책은 아키텍처에서 선택.

---

## 2. 스크리브너 장점 요약

| 장점 | StoryBuilder 대응 |
|------|-------------------|
| 비선형 집필 | 씬/회 단위 CRUD, 순서 변경 API |
| 통계·목표 | 집필 대시보드, 목표 단어 수, 진행률 (MVP Phase 3) |
| 스냅샷 | Manuscript 버전, 자동저장 이력 |

---

## 3. 서비스 기획·엔지니어링 관점

### 스크리브너의 한계

- **높은 학습 곡선**
- **클라우드·동기화 미지원** → 협업·실시간 공동 집필 불가

### StoryBuilder 차별화

- **DB 설계**: 글을 **블록/장면 단위**로 저장 (Scene + Manuscript). 순서 변경 시 `order_index`만 갱신하는 API로 바인더 드래그 지원.
- **협업·실시간**: WebSocket + Redis (또는 OT/CRDT)로 실시간 공동 집필 — 스크리브너가 하지 못하는 영역.

---

## 4. Cursor & 규칙 연계

에디터·바인더·아카이브 구현 시 참고:

1. **기능 요구**: "스크리브너 바인더처럼 사이드바에서 드래그로 순서를 바꾸는 기능. `backend/api`에 reorder 엔드포인트 설계해 줘."
2. **기술 검토**: "대용량 텍스트 변경 이력(스냅샷)을 Redis로 할지, DB JSON/Manuscript 버전으로 할지 검토해 줘."
3. **커밋**: `.cursor/rules/commit-conventions.mdc`에 따라 `feat(web): implement drag-and-drop binder` 형태로 PR/커밋 메시지 작성.

---

**정리**: 스크리브너의 바인더·코르크보드·스플릿·컴파일·스냅샷을 벤치마킹하여, StoryBuilder는 **소설/시나리오 분류**, **등장인물·로그라인·줄거리 아카이브**, **씬/회별 CRUD 및 순서 변경**으로 동일한 가치를 웹·협업 친화적으로 제공한다.
